## Objective
Establish the IMAP checking framework using `imapflow` on a recurring background loop.

## Tasks

<task type="auto">
  <name>Task 1: IMAP Worker Setup</name>
  <files>package.json, src/lib/queue.ts, worker.ts</files>
  <read_first>worker.ts, src/lib/queue.ts</read_first>
  <action>
    - `npm install imapflow` and `npm install -D @types/imapflow` or `mailparser`. We'll just use basic text matching for now, so `imapflow` and `mailparser` are needed. run `npm install imapflow mailparser; npm install -D @types/mailparser`.
    - In `src/lib/queue.ts`, add a recurring queue: `export const imapQueue = new Queue('imap-polling', { connection });`.
    - Also in `src/lib/queue.ts`, set it to run on boot: `imapQueue.add('poll', {}, { repeat: { every: 300000 } })` (Every 5 mins).
    - In `worker.ts`, add a new Worker for `'imap-polling'`. It fetches `SmtpSettings` for the first available user (since this is a single-tenant dev setup right now, or loops all users with SMTP settings). For each user:
    - Connect to IMAP (Host is usually `imap.gmail.com` if `smtp.gmail.com`, port 993, TLS).
    - Search for `UNSEEN` messages.
    - Extract text body using `mailparser`.
    - Mark message as read.
  </action>
  <acceptance_criteria>
    - `imapflow` and `mailparser` are integrated
    - The queue triggers parsing of UNSEEN messages
  </acceptance_criteria>
</task>
