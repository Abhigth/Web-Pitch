## Objective
Process the extracted IMAP bodies, match them against Leads, and dispatch the automated follow-up.

## Tasks

<task type="auto">
  <name>Task 1: Reply Evaluation & Follow-Up Dispatch</name>
  <files>worker.ts</files>
  <read_first>worker.ts, src/lib/queue.ts, prisma/schema.prisma</read_first>
  <action>
    - Inside the `imap-polling` worker loop (handling the parsed `mailparser` email object):
    - Extra the sender's email address.
    - `const lead = await prisma.lead.findFirst({ where: { email: senderEmail } })`.
    - If `!lead`, skip.
    - Perform a basic positive intent keyword regex on the email body text (`/yes|sure|interested|send|pricing|details|how much|love to/i`).
    - If positive, update status: `prisma.lead.update({ where: { id: lead.id }, data: { status: 'REPLIED_POSITIVE' } })`.
    - Import `emailQueue` and enqueue the follow up: `await emailQueue.add('sendEmail', { leadId: lead.id, templateText: "Awesome, here is the link to my website plans: https://my-portfolio.com. Let me know which one works for you!" })`.
    - If negative (`/no|stop|unsubscribe|not interested/i`), update status to `REPLIED_NEGATIVE`.
  </action>
  <acceptance_criteria>
    - Incoming emails map to Leads
    - Basic Keyword Intent updates the Lead status
    - Positive intent automatically enqueues a follow-up email job
  </acceptance_criteria>
</task>
