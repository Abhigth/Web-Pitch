## Objective
Establish the background job queue infrastructure using BullMQ and Redis, keeping exact synchronization with the Next.js Campaign creation flow.

## Tasks

<task type="auto">
  <name>Task 1: Queue Installation and Configuration</name>
  <files>package.json, src/lib/queue.ts, worker.ts</files>
  <read_first>src/app/dashboard/campaigns/page.tsx</read_first>
  <action>
    - Install `bullmq` and `ioredis` dependencies using npm. (Also install `tsx` as a devDependency to comfortably run the worker in typescript context).
    - Create `src/lib/queue.ts`. Inside, import `Queue` from `bullmq`. Create a Redis connection using `process.env.REDIS_URL || 'redis://localhost:6379'`. Export `const scrapeQueue = new Queue('scrape-campaigns', { connection })`.
    - Create `worker.ts` in the root directory. Inside, instantiate a `Worker` on the `scrape-campaigns` queue. Connect to the same Redis URL.
    - Inside the worker's processing flow, just `console.log('Processing job for campaign:', job.data.campaignId)`.
    - Once the job is processed, update the campaign status to 'COMPLETED' using Prisma: `prisma.campaign.update({ where: { id: job.data.campaignId }, data: { status: 'COMPLETED' } })`.
  </action>
  <acceptance_criteria>
    - `src/lib/queue.ts` exists and exports `scrapeQueue`
    - `worker.ts` exists and connects to Redis to process `scrape-campaigns`
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Task 2: Queue Integration in Campaign Creation</name>
  <files>src/app/dashboard/campaigns/page.tsx</files>
  <read_first>src/app/dashboard/campaigns/page.tsx, src/lib/queue.ts</read_first>
  <action>
    - Modify the server action `createCampaign` within `src/app/dashboard/campaigns/page.tsx`.
    - Import `scrapeQueue` from `@/lib/queue`.
    - When calling `prisma.campaign.create`, set the `status` string to `'SCRAPING'` immediately instead of the DB default 'DRAFT'.
    - After Prisma creates the campaign record, add a job to the queue: `await scrapeQueue.add('scrape', { campaignId: campaign.id, location: campaign.location, keywords: campaign.keywords })`.
  </action>
  <acceptance_criteria>
    - `src/app/dashboard/campaigns/page.tsx` imports `scrapeQueue`
    - `scrapeQueue.add` is called in `createCampaign`
    - newly created campaigns have the status `'SCRAPING'`
  </acceptance_criteria>
</task>
