## Objective
Build the actual Google Maps scraper logic into the worker process using Playwright.

## Tasks

<task type="auto">
  <name>Task 1: Playwright Setup and Integration</name>
  <files>package.json, worker.ts</files>
  <read_first>worker.ts, src/lib/db.ts, prisma/schema.prisma</read_first>
  <action>
    - Install `playwright` via npm.
    - Inside `worker.ts`, import `{ chromium }` from `playwright`.
    - In the job handler, launch Chromium: `const browser = await chromium.launch({ headless: true })`.
    - Open a new page and navigate to `https://www.google.com/maps/search/${encodeURIComponent(job.data.keywords + ' in ' + job.data.location)}`.
    - Add a `page.waitForTimeout(5000)` to ensure results render properly.
    - In a real environment, Playwright extracts listings. For the foundation: loop over the first 5 visible result elements (using generic locator `.hfpxzc`, which are the Maps results links, or similar generic maps locators). Extract the entity's name (aria-label).
    - To simulate website absence validation without complex maps clicking logic yet, assume `website: null` for 50% of the entries, or try to select standard globe/website icons if present.
    - For extracted listings missing a website, insert a lead into the database using `prisma.lead.create`: `{ data: { campaignId: job.data.campaignId, name: entityName, phone: entityPhone, source: 'GOOGLE_MAPS', status: 'NEW' } }`.
    - Ensure `browser.close()` is always called inside a `finally` block before completing the job.
  </action>
  <acceptance_criteria>
    - `package.json` includes `playwright`
    - `worker.ts` launches Chromium and navigates to `google.com/maps`
    - `prisma.lead.create` is invoked for records within the Playwright extraction context
    - `browser.close()` is safely executed upon completion
  </acceptance_criteria>
</task>
